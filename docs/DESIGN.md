# Design Overview

ULTSNOWER 采用轻量 Web 服务架构，核心目标是让商户能够独立管理客户、余额、充值、订单和后台运营配置。

## 模块

- **Web App**：提供客户侧页面、商户后台页面和 JSON API。
- **Service Layer**：集中处理客户、余额、充值卡、订单、管理员、审计日志和系统设置。
- **Database Layer**：使用 SQLite 保存业务数据，便于本地开发、演示和小规模部署。
- **Security Utilities**：负责密码哈希、会话 token、验证码、输入校验和权限检查。

## 数据边界

本服务保存以下业务数据：

- 客户账户与密码哈希。
- 客户余额与充值记录。
- 充值卡及其使用状态。
- 本地订单与订单历史。
- 管理员账户、角色与审计日志。
- 系统公告、维护模式、隐私模式等运营设置。

## 运行模式

- 默认使用 SQLite，数据库路径由 `MERCHANT_DB_PATH` 指定。
- 默认监听 `127.0.0.1:8020`，可通过 `MERCHANT_HOST` 和 `MERCHANT_PORT` 修改。
- 生产部署建议放在可信反向代理之后，并启用 HTTPS。

## 安全原则

- 不提交 `.env`、数据库、日志或真实凭据。
- 生产环境必须覆盖默认管理员密码。
- 管理员敏感操作写入审计日志。
- 客户密码只保存哈希，不明文保存。
- 内部任务接口如需跨主机调用，必须配置强随机 token。
